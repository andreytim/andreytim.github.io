Number of sub-trees in a tree

November 28, 2014

Reading time ~4 minutes

Intro

A couple of days ago I came across an interesting problem on TopCoder - CentaurCompanyDiv2. What it asks is, basically, to find the number of sub-trees of a given tree. I am going to show how to do it in Java using dynamic programming approach and also without recursion. Check it out if interested!

Details

For a start, let’s discuss a general approach. As you know, a tree is a recursive data structure and, actually, a huge amount of trees-related problems could be easily solved through recursion. So, thinking about formulating the solution in that way is always a good idea while working with any trees. Well, let’s imagine that we are at some node N and we know the number of sub-trees for all trees rooted in its children. How can we express the number of subtrees for the tree rooted in our current node N (let’s call this tree T(N))?
First, we must gather all numbers of sub-trees from our children, because any sub-tree there is also a sub-tree for our T(N).
Second, each sub-tree of child C1 rooted in C1 could be connected through N to any sub-tree of any other child C2 rooted in C2. Finally, we add our root N as a distinct sub-tree.
Thus, we got the following formula:

S(N) = sum(S(Ci)) + sum(SR(Ci)`SR(Cj)) + 1**, where

S(N) - number of sub-trees for the tree rooted in our N,
sum(S(Ci)) - sum of numbers of sub-trees for the children of N,
sum(SR(Ci)`SR(Cj))* - sum of all combinations of sub-trees rooted in children of N.

Now, let’s think about how to implement this approach.
First, let’s for simplicity assume that the vertices of an input tree or graph are numbers from 1 to N (like in the TopCoder problem above).
Second, we can see that it doesn’t matter which vertex to pick as an initial root, so lets pick the vertex 1.
Third, as we should know the solution for all children of a current node at any step we must go from the bottom to the top of our tree. That’s actually an order of backtracking from depth-first search traversal.
Fourth, as our vertices are positive integers we can store the pre-calculated solutions in an array of integers. Also we need two keep track of two arrays - number of sub-trees, number of rooted sub-trees for each node.
Fifth, to avoid recursion we could put all vertices to a stack during the DFS traversal and than use it as an ordered list for a main-loop.

So, let’s finally look at the Java code: 

public long count(Map<Integer, Set<Integer>> graph) {
    long[] numST = new long[graph.size()+1];
    long[] numRootedST = new long[graph.size()+1];
    for (Integer v : getBacktrackDfsOrder(graph)) {
        long st = 1, rst = 1;
        for (Integer n : graph.get(v)) {
            st += numST[n] + rst*numRootedST[n];
            rst += rst*numRootedST[n];
        }
        numST[v] = st;
        numRootedST[v] = rst;
    }
    return numST[1];
}

As you can see, it’s pretty simple and concise. Here is also the code of an auxiliary method for getting the “backtrack-DFS” order: 

private Iterable<Integer> getBacktrackDfsOrder(Map<Integer, Set<Integer>> graph) {
    Deque<Integer> order = new LinkedList<>();
    Deque<Integer> dfs = new LinkedList<>();
    Set<Integer> visited = new HashSet<>();
    dfs.push(1); visited.add(1);
    while (!dfs.isEmpty()) {
        Integer curr = dfs.pop();
        order.push(curr);
        for (Integer next : graph.get(curr)) {
            if (!visited.contains(next)) {
                visited.add(next);
                dfs.push(next);
            }
        }
    }
    return order;
}

Just DFS, basically.
